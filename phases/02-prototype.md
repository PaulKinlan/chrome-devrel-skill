# Phase 02 — Prototype

## Entry

Problem/use cases are sufficiently understood to test an API/design behind a runtime flag or experimental build; Intent to Prototype/public design work may begin.

## Decisions

- Can the proposed design solve realistic tasks ergonomically and safely?
- Does it compose with frameworks, libraries, servers, deployment, permissions, lifecycle, and fallbacks?
- Which risks/questions need implementation evidence?
- Is it ready for representative developers to try?

## Work and evidence

- Build the smallest realistic implementation and task-based samples, not only API-unit demos.
- Run friction logs from clean setup through framework/server/mobile/desktop, failure, recovery, cleanup, console/network, accessibility, performance, and visual behavior.
- Compare explainer/spec/docs/sample/implementation and record divergence.
- Prototype alternatives where design decisions remain uncertain.
- Engage framework/tool authors and candidate users; classify evidence honestly.
- Start test strategy, feature detection/fallback, privacy/security/accessibility pre-review, and other-engine/standards engagement.

## Artifacts

- Updated explainer and API examples
- Prototype/demo inventory with stable routes/IDs
- Integration spikes and compatibility matrix
- Friction/evidence report
- Open design decision log
- Developer-trial plan, support route, feedback instrument, and readiness state

## Common failures

- Polished demo masking hidden setup or resource leaks
- Desktop-only evidence presented as platform support
- Framework compatibility assumed from vanilla JavaScript
- Feedback gathered without explicit learning questions

## Transition packet: prototype → developer trials

Working implementation/build instructions; tested tasks/environments; known limitations/fallbacks; explainer/spec deltas; friction findings; review risks; integration results; candidate cohort; trial learning questions, feedback routes, support/triage plan, evidence collection, exit/redesign criteria, and unresolved blockers.
