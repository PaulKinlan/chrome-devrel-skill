# Phase 01 — Incubation

## Entry

A problem/opportunity has been identified; use cases, alternatives, and solution shape are still open. A ChromeStatus entry or public explainer may be starting.

## Decisions

- Is the developer/end-user problem real, important, and recurring?
- What jobs, constraints, and outcomes define success?
- What existing web/native/server/library solutions exist and why are they insufficient?
- What is the narrowest useful browser intervention?
- Who is affected and who should co-design it?

## Work and evidence

- Run ecosystem/customer research across web communities, frameworks, public support/issues, native platforms, mini-apps, chat/LLM/agent ecosystems, and existing workarounds.
- Seek contradictory evidence and answer “why not?”
- Interview representative developers/users; distinguish anecdotes, surveys, usage data, and commitments.
- Write use cases before API details; include non-goals and alternatives.
- Start stakeholder/user-impact critique, abuse/threat model, interoperability path, and implementation feasibility.
- Establish public incubation venue and evidence ledger.

## Artifacts

- Problem/jobs/use-case brief and segment map
- Alternative/adjacent-platform analysis
- Developer research plan and evidence ledger
- Initial explainer critique/draft with goals, non-goals, alternatives, privacy/security/accessibility considerations
- Candidate co-designers/trial partners with evidence stage
- Prototype questions and learning goals

## Common failures

- Searching only for support, not counterevidence
- Using broad category interest as API demand
- Copying a native API without adapting to web principles
- Naming customers who have not consented or treating interest as commitment
- Locking the API shape before understanding integration

## Transition packet: incubation → prototype

Validated/contested jobs and use cases; segment/community map; current workarounds and costs; adjacent-platform patterns; alternatives rejected with evidence; affected users/stakeholders; candidate partners; initial review risks; explainer/source links; prototype hypotheses, tasks, environments, measures, and stop/redesign conditions.
